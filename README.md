# บันทึกประวัติการสนทนา LINE OA ลงตารางอัตโนมัติ

ทุกครั้งที่ลูกค้าทักแชทเข้ามาทาง **LINE Official Account** ระบบจะ **บันทึกประวัติการสนทนาลงตารางให้อัตโนมัติ**
— เก็บ **ชื่อผู้ส่ง · ข้อความ · เวลา** ไว้ครบทุกข้อความ และแสดงบนหน้าเว็บที่อัปเดตสดแบบเรียลไทม์
คุณจึงย้อนดูได้ว่า *ใครทักมา · พิมพ์ว่าอะไร · ตอนกี่โมง* โดยไม่ต้องเปิดแอป LINE ไล่อ่านทีละแชท

### ตัวอย่างตารางที่ระบบบันทึกให้

| ID | ชื่อผู้ใช้ | ประเภท | ข้อความ | เวลาที่ได้รับ |
|----|-----------|---------|---------|----------------|
| 7  | คุณสมหญิง  | sticker | —                                   | 2026-06-24 14:33:20 |
| 6  | คุณสมหญิง  | image   | —                                   | 2026-06-24 14:32:45 |
| 5  | คุณสมหญิง  | text    | ขอบคุณค่ะ เดี๋ยวโอนแล้วส่งสลิปให้นะคะ | 2026-06-24 14:32:07 |
| 4  | คุณวิชัย   | file    | —                                   | 2026-06-24 14:19:10 |

| 2  | คุณวิชัย   | text    | สวัสดีครับ สนใจสินค้าตัวนี้ครับ      | 2026-06-24 14:17:12 |

> **คอลัมน์ประเภท** ใช้ค่าตามจริงจาก LINE — `text` (ตัวอักษร) · `image` (รูปภาพ) · `sticker` (สติกเกอร์) · `file` (ไฟล์) ฯลฯ
> สำหรับข้อความที่ไม่ใช่ตัวอักษร (รูป/สติกเกอร์/ไฟล์) ระบบจะบันทึก **ประเภทและเวลา** ไว้ ส่วนช่องข้อความจะว่าง (`—`)
>
> ตารางจริงบนหน้าเว็บจะ **เรียงข้อความล่าสุดไว้บนสุด** และ **รีเฟรชอัตโนมัติทุก 2 วินาที**
> — ลูกค้าพิมพ์เข้ามาปุ๊บ ข้อความก็โผล่ในตารางทันที ไม่ต้องกดรีเฟรชเอง

## ระบบนี้ช่วยอะไรคุณ

- 🗂️ **มีประวัติการสนทนาเก็บไว้เอง** — ทุกข้อความถูกบันทึกลงฐานข้อมูลของคุณ ไม่หายแม้ปิดแชทหรือรีสตาร์ตเครื่อง
- 👀 **เห็นทุกแชทในที่เดียว** — ดูข้อความจากลูกค้าทุกคนรวมกันในตารางเดียว แทนการเปิดทีละห้องแชท
- 🔎 **ย้อนดู / ค้นหาได้ภายหลัง** — มีชื่อ ข้อความ และเวลาครบ ใช้ตรวจสอบหรือทำรายงานยอดสั่งซื้อได้
- ⚡ **อัปเดตสดแบบเรียลไทม์** — ข้อความใหม่ขึ้นตารางเองภายในไม่กี่วินาที
- 💬 **ตอบกลับลูกค้าได้** — ตอบกลับอัตโนมัติ และมีช่องส่งข้อความหาลูกค้าได้จากหน้าเว็บ (ดูหัวข้อถัดไป)

## หน้าจอใช้งาน

เปิดเว็บเบราว์เซอร์ไปที่หน้าแดชบอร์ด แล้วคุณจะเห็น 2 ส่วนหลัก:

1. **ตารางประวัติการสนทนา** — รายการข้อความทั้งหมด (ชื่อผู้ใช้ · ประเภท · ข้อความ · เวลา) อัปเดตสดทุก 2 วินาที
2. **ช่องส่งข้อความ (Push)** — พิมพ์ส่งข้อความหาลูกค้าได้โดยตรงจากหน้าเว็บ (ใส่รหัสผู้ใช้ + ข้อความ แล้วกดส่ง)

> 💡 **ชื่อผู้ใช้** จะดึงมาจากโปรไฟล์ LINE ของลูกค้าโดยอัตโนมัติเมื่อทักเข้ามาครั้งแรก
> **ประเภท** แสดงเป็นค่าจริงจาก LINE เช่น `text`, `image`, `sticker`, `file` และ **เวลา** บันทึกเป็นเวลามาตรฐานสากล (UTC)

## ความสามารถเรื่องการตอบกลับ

- 🤖 **ตอบกลับอัตโนมัติ (auto-reply)** — ระบบตอบกลับข้อความที่ลูกค้าส่งมาให้ทันที (โหมดตัวอย่างจะตอบแบบสะท้อนข้อความเดิม)
- 📤 **ส่งข้อความหาลูกค้า (push)** — พิมพ์ส่งข้อความเองได้จากหน้าแดชบอร์ด
- 🧾 ข้อความที่ระบบส่งออกไป ก็ถูก **บันทึกเก็บไว้** เช่นกัน

## 🧱 สถาปัตยกรรม (Architecture)

```
        LINE Platform
             │  (POST + X-Line-Signature)
             ▼
┌─────────────────────────────┐        ┌──────────────────────────────┐
│  Webhook (FastAPI)  :9990    │        │   Dashboard (NiceGUI)  :9991  │
│  /line/webhook               │        │   ตารางข้อความ + ปุ่ม Push      │
│  - ตรวจลายเซ็น               │        │   - refresh ทุก 2 วินาที        │
│  - บันทึกข้อมูล              │        └───────────────┬──────────────┘
│  - ตอบกลับ (echo)            │                        │
└──────────────┬──────────────┘                        │
               └──────────────┬─────────────────────────┘
                              ▼
                       SQLite  (./data/line.db)
```

ทั้งสองพอร์ตรันใน **โปรเซสเดียว / คอนเทนเนอร์เดียว** (uvicorn 2 ตัวผ่าน `asyncio.gather` ใน `src/main.py`)

| พอร์ต | บริการ | รายละเอียด |
|------|--------|-----------|
| **9990** | Webhook (FastAPI) | นำ URL นี้ไปตั้งค่าใน LINE Developers Console (`POST /line/webhook`) |
| **9991** | Dashboard (NiceGUI) | เปิดดูข้อความในเบราว์เซอร์ (`GET /`) |

### ความสามารถเชิงเทคนิค

- 📥 **รับ Webhook จาก LINE** — ปลายทาง FastAPI ที่ `POST /line/webhook`
- 🔐 **ตรวจสอบลายเซ็น** `X-Line-Signature` (HMAC-SHA256 จาก raw body) ตามมาตรฐานของ LINE
- 🗄️ **บันทึกข้อมูลลง SQLite** — เก็บผู้ใช้ (users), เหตุการณ์ดิบ (events), ข้อความ (messages) และการตอบกลับ (replies)
- 📊 **แดชบอร์ดเรียลไทม์** — ตารางข้อความสร้างด้วย NiceGUI รีเฟรชอัตโนมัติทุก 2 วินาที
- 🔒 **เก็บความลับไว้ใน `.env` เท่านั้น** (repo นี้เป็น public — มีแต่ `.env.example` ที่ถูก commit)

## 🚀 เริ่มต้นใช้งาน (Getting started)

### 1) เตรียมความลับใน `.env`

คัดลอกไฟล์ตัวอย่างแล้วใส่ค่าจริงจาก **LINE Developers Console**:

```bash
cp .env.example .env
```

แก้ไขค่าต่อไปนี้ในไฟล์ `.env` (ห้าม commit ไฟล์นี้):

```dotenv
LINE_CHANNEL_SECRET=<channel secret ของคุณ>
LINE_CHANNEL_ACCESS_TOKEN=<channel access token ของคุณ>
```

> ⚠️ **ข้อควรระวังด้านความปลอดภัย:** repo นี้เป็น public — `.env`, `data/`, และ `*.db`
> ถูกใส่ไว้ใน `.gitignore` แล้ว ห้ามนำ token/secret ขึ้น GitHub โดยเด็ดขาด

### 2) รันแบบโลคอลด้วย uv

```bash
uv sync --all-groups          # ติดตั้ง dependencies
uv run python -m src.main     # รัน webhook (:9990) + dashboard (:9991)
```

เปิดแดชบอร์ดที่ <http://localhost:9991>

### 3) รันด้วย Docker

```bash
docker compose up --build
```

- Webhook: <http://localhost:9990/line/webhook>
- Dashboard: <http://localhost:9991>
- ฐานข้อมูลถูกเก็บไว้ที่โฟลเดอร์ `./data` (ผ่าน volume) จึงไม่หายเมื่อ restart

## 🔗 เชื่อมต่อกับ LINE จริง

LINE ต้องการ URL แบบ **HTTPS สาธารณะ** จึงต้องเปิด tunnel มาที่พอร์ต webhook (9990):

```bash
cloudflared tunnel --url http://localhost:9990
# หรือ
ngrok http 9990
```

จากนั้นนำ URL ไปตั้งใน LINE Developers Console:

```
Webhook URL:  https://<โดเมน-tunnel>/line/webhook
```

แล้วกด **Verify** และเปิด **Use webhook** ให้เรียบร้อย

## 🧪 ทดสอบโดยไม่ต้องต่อ LINE

มีสคริปต์สำหรับยิง webhook ปลอม (ลงลายเซ็นถูกต้องด้วย channel secret ของคุณ):

```bash
# เทอร์มินัล 1 — รันแบบ offline (ปิดการตอบกลับ เพราะ reply token เป็นของปลอม)
AUTO_REPLY=false uv run python -m src.main

# เทอร์มินัล 2
uv run python scripts/send_test_webhook.py "สวัสดีครับ"
```

ข้อความจะปรากฏในตารางแดชบอร์ดภายใน ~2 วินาที

## 📁 โครงสร้างโปรเจกต์ (Project structure)

```
src/
  main.py            # entrypoint: uvicorn 2 ตัว (webhook :9990, dashboard :9991)

    webhook.py       # ตรวจลายเซ็น + parse + บันทึก + ตอบกลับ

  dashboard/
    app.py           # FastAPI + NiceGUI (ui.run_with)
    pages.py         # ตารางข้อความ + ฟอร์ม push
scripts/
  send_test_webhook.py   # ยิง webhook ตัวอย่างที่ลงลายเซ็นแล้ว
tests/                   # signature / persist / services / client
```
